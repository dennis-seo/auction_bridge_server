시스템/내부 (Internal)

Cloud Scheduler·헬스체크 등 내부 트리거
언제 쓰나요? Cloud Run 컨테이너 헬스체크, Cloud Scheduler가 OIDC 토큰으로 데일리 cron을 호출할 때
POST /api/v1/internal/cron/daily-ingest 🔑 OIDC 필요
일 1회 온비드 ingest (Cloud Scheduler 호출)

Cloud Scheduler가 OIDC ID 토큰(Authorization: Bearer)을 붙여 호출하는 내부 엔드포인트. 토큰의 audience가 settings.CRON_AUDIENCE(=Cloud Run service URL)와 같고, 발급자 이메일이 CRON_SERVICE_ACCOUNT_EMAIL과 정확히 일치해야 통과한다. ingest → 입찰결과 #8 보강 → #9 fallback → 부동산 사진 → 동산 사진 순으로 실행한다.
